I'm looking to write a skill file that can be used by gemini to help run serial commands on a fuchsia device. This skill should go into the "src/devices/skills/run_on_serial/SKILL.md" file. You can override this file if it already exists.

There is a script that can be used to run serial commands on a fuchsia device, and observe its output. It can be found at "src/devices/skill_scripts/run_serial_cmd.sh".

You would use this script to run serial commands on a fuchsia device. The script will use the environment variables FUCHSIA_SERIAL_UNIX_SOCKET and FUCHSIA_SERIAL_LOG_FILE to find the serial socket and log file. If those environment variables are not set, ask the user to set them.

The script accepts a single argument, which is the command to run on the serial console. For example, if you want to run the "driver list" command, you would run:

run_serial_cmd.sh "driver list"

Note that the serial output may contain intertwined output from the device, which may include generic logs from the device, along with the output from the command you invoked. Use your judgement to determine which lines you care about and which ones to ignore. For example any line that contains a timestamp and pid/tid (eg: "[24462.203] 234251:234253>") is likely generic logs from the device, while a line that contains the output of the command you invoked is likely what you need.
