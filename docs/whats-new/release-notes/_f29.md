**Version:** F29

## Audio

The following platform updates have been made for audio in Smart Displays:

*   **Expanded AudioCore and Driver Inspect observability.** Inspect data now
    includes `RoleManager` errors for better diagnostics of past failures,
    tracks min/max/sum/overrun/underrun metrics per ring buffer, and retains
    recent task records.

    *   Changes:
        [fxr/1385750](https://fuchsia-review.googlesource.com/c/fuchsia/+/1385750),
    [fxr/1366956](https://fuchsia-review.googlesource.com/c/fuchsia/+/1366956),
    [fxr/1361564](https://fuchsia-review.googlesource.com/c/fuchsia/+/1361564)

The following platform updates have been made for other products:

*   **Enhanced Audio Device Registry (ADR) performance and reliability.** The
    ADR thread now utilizes a scheduler profile to improve stream establishment
    performance under load. Additionally, ADR now supports topologies without
    ring buffers, retries on transient ring buffer creation failures, and
    correctly accounts for driver overallocation in ring buffer memory
    reporting.

    *   Changes:
        [fxr/1384433](https://fuchsia-review.googlesource.com/c/fuchsia/+/1384433),
    [fxr/1364452](https://fuchsia-review.googlesource.com/c/fuchsia/+/1364452),
    [fxr/1335114](https://fuchsia-review.googlesource.com/c/fuchsia/+/1335114),
    [fxr/1329159](https://fuchsia-review.googlesource.com/c/fuchsia/+/1329159)

*   **Improved `ffx audio` tooling capabilities.** The `ffx audio device play`
    command now uses scheduler profiles and larger ring buffers to enable
    glitch-free playback validation. The tool also now displays the ADR
    `token_id` in device lists and includes a fix for calculation underflows.

    *   Changes:
        [fxr/1374081](https://fuchsia-review.googlesource.com/c/fuchsia/+/1374081),
    [fxr/1336967](https://fuchsia-review.googlesource.com/c/fuchsia/+/1336967),
    [fxr/1346660](https://fuchsia-review.googlesource.com/c/fuchsia/+/1346660)

*   **Updated Signal Processing topology rules.** The SDK now allows
    `DAI_INTERCONNECT` elements to be self-referential within a topology,
    enabling support for standalone DAI configurations.

    *   Changes: [fxr/1348753](https://fuchsia-review.googlesource.com/c/fuchsia/+/1348753)

## Cobalt

The following updates have been made for Cobalt:

*   **Java metric ID type update** The Java source generator now declares
    `uint32` metric IDs as `int` instead of `long`. This aligns with the Cobalt
    logging API and removes the need for explicit casts in client code.

    *   Changes: [fxr/1368858](https://fuchsia-review.googlesource.com/c/cobalt/+/1368858)

*   **Event Logger validation update** The Event Logger now accepts histograms
    containing zero counts for impossible buckets. This prevents the rejection
    of dense vectors generated by Samplers where impossible buckets are included
    with a zero count.

    *   Changes: [fxr/1366636](https://fuchsia-review.googlesource.com/c/cobalt/+/1366636)
